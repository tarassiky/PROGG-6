
# Лабораторная работа №2: Методы оптимизации вычислений

## Цель работы
Исследовать методы оптимизации вычислений на примере численного интегрирования методом прямоугольников. Сравнить производительность:
- чистой реализации на Python;
- с использованием потоков (`ThreadPoolExecutor`);
- с использованием процессов (`ProcessPoolExecutor`);
- с компиляцией Cython (с GIL и с отпусканием GIL);
- с комбинацией Cython + nogil + потоки.

## Требования

- Python 3.12 или 3.13
- Установленный Cython: `pip install cython`
- Компилятор C++ (для Windows: Microsoft Visual C++ 14.0+; для Linux: gcc)

## Запуск итераций

### Итерация 1 – базовая функция integrate
```bash
python iteration1_integrate.py
```
Выполняет doctest, unittest и замер времени для 1 млн итераций.

### Итерация 2 – потоки (ThreadPoolExecutor)
```bash
python iteration2_threads.py
```
Из-за GIL ускорения нет.

### Итерация 3 – процессы (ProcessPoolExecutor)
```bash
python iteration3_processes.py
```
Ускорение на многоядерном CPU.

### Итерация 4 – Cython (без nogil)
```bash
cd iteration4_cython
python setup.py build_ext --inplace
python run_benchmark.py
```
Ускорение отсутствует, если внутри цикла вызывается Python-функция.

### Итерация 5 – Cython + nogil + потоки
```bash
cd iteration5_nogil
python setup.py build_ext --inplace
python run_benchmark.py
```
Реальное ускорение за счёт параллельной работы потоков без GIL.

## Аннотированный HTML-отчёт Cython

После компиляции в папках `iteration4_cython` и `iteration5_nogil` появляются файлы `integrate_cy.html` и `integrate_nogil.html`. Откройте их в браузере.  
- Жёлтый фон строки означает взаимодействие с Python API.  
- В `integrate_nogil` внутри цикла `with nogil` жёлтых строк быть не должно – признак эффективной оптимизации.

---

## Выводы

1. **Потоки** в стандартном Python не ускоряют CPU‑интенсивные задачи из-за GIL.
2. **Процессы** дают хорошее ускорение, но имеют накладные расходы.
3. **Cython** без отпускания GIL неэффективен, если внутри цикла вызываются Python-функции.
4. **Cython + nogil** позволяет писать параллельные вычисления на потоках, обходя GIL.
5. Для максимальной производительности целесообразно комбинировать Cython с `nogil` и потоками.

---

## Ответы на дополнительные вопросы

**Примитивы синхронизации (мьютекс, семафор)** – в данной задаче не нужны, потому что каждый поток/процесс работает с независимыми данными (своя часть отрезка, своя переменная `acc`). Глобальные ресурсы не модифицируются конкурентно.

**Python 3.14 и GIL** – начиная с версии 3.13 доступна опциональная сборка без GIL (PEP 703). В такой сборке стандартные потоки начнут параллельно выполнять CPU‑задачи, и отпадает необходимость в ручном отпускании GIL через Cython.
