## «Асинхронное программирование на Python с применением Flask и asyncio»

## 1. Цель работы

Разработать веб-приложение на Flask, которое выполняет асинхронные запросы к API Crossref (https://api.crossref.org) для поиска научных публикаций по автору и/или названию. Приложение должно демонстрировать преимущества асинхронного ввода-вывода при работе с внешними HTTP-сервисами.

---

## 2. Задачи

1. Создать HTML-форму с двумя полями (автор, название) на Bootstrap.
2. Реализовать асинхронный HTTP-клиент на базе `aiohttp` для обращения к Crossref API.
3. Извлечь из JSON-ответа следующие поля: название статьи, первый автор, журнал, год, аффилиация.
4. Поддержать параллельный поиск при вводе нескольких авторов через запятую.
5. Добавить обработку ошибок (таймауты, пустые ответы, ошибки соединения).
6. Оформить результаты в виде карточек с адаптивной сеткой Bootstrap.
7. Снабдить код комментариями, поясняющими работу асинхронных конструкций.

## 4. Архитектура приложения

Flask является синхронным WSGI-фреймворком. Для использования `asyncio` внутри запроса создаётся отдельный цикл событий с помощью функции `run_async()`. Все запросы к Crossref API выполняются в асинхронных корутинах, что позволяет:

- Не блокировать поток Flask на время ожидания ответа от внешнего API.
- Параллельно отправлять несколько запросов (например, при поиске по нескольким авторам).

**Схема работы:**
Пользователь → Форма (POST) → Маршрут Flask
→ run_async(корутина) → aiohttp.ClientSession
→ параллельные запросы к Crossref → парсинг ответа
→ отображение через render_template()

## 5. Обработка ошибок

| Ситуация | Действие приложения |
|----------|---------------------|
| Нет интернета | Выводит сообщение: «Ошибка при обращении к API Crossref» |
| Таймаут (>10 сек) | Логирует ошибку, возвращает пустой список и сообщение «Ничего не найдено» |
| Пустое поле автор+название | Валидация на стороне сервера: «Укажите хотя бы автора или название» |
| API вернул 429 (Too Many Requests) | Логирует код, результат – пустой список |

## 6. Объяснение асинхронных конструкций

### 6.1. `async def fetch_publications(...)`

- Использует `async with aiohttp.ClientSession()` – контекстный менеджер для управления подключениями.
- `await session.get(...)` – не блокирует поток, управление возвращается в цикл событий до получения ответа.
- `await resp.json()` – асинхронное чтение тела ответа.

### 6.2. `asyncio.gather(*tasks)`

Позволяет запустить несколько корутин **параллельно** в одном цикле событий. В нашем случае для каждого автора создаётся задача `fetch_publications`, и все они выполняются одновременно.

### 6.3. `run_async(coro)`

Функция-обёртка, необходимая из-за синхронной природы Flask. Создаёт новый цикл событий `asyncio.new_event_loop()`, запускает в нём корутину и возвращает результат. После выполнения цикл закрывается.

**Почему нельзя просто `asyncio.run()`?**  
`asyncio.run()` создаёт цикл и пытается его переиспользовать, но Flask в режиме отладки может вызывать многопоточность, что приводит к ошибкам. Поэтому мы создаём свежий цикл на каждый запрос.

## 7. Выводы

В ходе лабораторной работы было разработано веб-приложение, демонстрирующее:

- Практическое применение `asyncio` и `aiohttp` для неблокирующих HTTP-запросов.
- Возможность параллельного выполнения нескольких запросов с помощью `asyncio.gather`, что значительно ускоряет поиск при вводе нескольких авторов.
- Корректную обработку ошибок (таймауты, сетевые проблемы, пустые ответы).
- Интеграцию асинхронного кода с синхронным WSGI-фреймворком через создание временных циклов событий.

Приложение удовлетворяет всем техническим требованиям: используется Flask, Bootstrap, aiohttp, предусмотрен параллельный поиск, реализовано извлечение всех необходимых полей (с аффилиацией или фразой «Не указано»).
