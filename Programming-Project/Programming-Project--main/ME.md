

# ИПОТЕЧНЫЙ КОНСУЛЬТАНТ

## ML-сервис для оценки одобрения ипотеки

### Техническая документация
#### Frontend-часть проекта



## 1. Обзор проекта

Данный документ описывает frontend-часть сервиса «Ипотечный консультант» — веб-интерфейс, разработанный для взаимодействия с REST API сервиса. Интерфейс позволяет пользователю вводить данные клиента, получать предсказание о вероятности одобрения ипотечной заявки, а также загружать CSV-файлы для массового анализа.

### Роль в командном проекте

| **Роль** | **Участник** | **Задачи** |
|----------|--------------|------------|
| ML Engineer | — | Обучение модели, сериализация pipeline.pkl |
| Backend Developer | — | FastAPI-сервер, интеграция модели, API |
| **Frontend Developer** | **—** | **Веб-интерфейс, подключение к API, UX/UI** |

### Стек технологий

- **Язык разработки:** HTML5, CSS3, JavaScript (ES6+)
- **Стилизация:** CSS-переменные для поддержки тем, медиазапросы для адаптивности
- **Сетевые запросы:** Fetch API — нативный JavaScript-интерфейс для HTTP-запросов
- **Обработка файлов:** FileReader API — чтение CSV-файлов на стороне клиента
- **Хранение настроек:** localStorage — сохранение выбранной темы
- **Сервер разработки:** live-server — локальный сервер с автоматической перезагрузкой
- **Менеджер зависимостей:** отсутствует (чистый HTML/CSS/JS без фреймворков)

## 3. Описание файлов

### index.html — Главная страница

Главный HTML-документ, содержащий всю разметку интерфейса. Построен на семантических элементах с использованием технологии **Glassmorphism** (стеклянные карточки с размытием фона).

#### Структура страницы

1. **Header** — заголовок сервиса и переключатель темы
2. **Левая колонка** — форма для ввода данных одного клиента
3. **Правая колонка** — блок для загрузки CSV-файлов

#### Форма ввода

Содержит 13 полей, соответствующих признакам клиента из датасета:

| **Признак** | **Тип** | **Значения** |
|-------------|---------|--------------|
| person_age | number | 18–100 |
| person_gender | select | male, female |
| person_education | select | Bachelor, Master, PhD, High School, Associate |
| person_income | number | ≥ 0 |
| person_emp_exp | number | ≥ 0 |
| person_home_ownership | select | rent, own, mortgage, other |
| loan_amnt | number | ≥ 0 |
| loan_intent | select | homeimprovement, debtconsolidation, medical, education, majorpurchase, smallbusiness |
| loan_int_rate | number | ≥ 0 |
| loan_percent_income | number | автоматически вычисляется (readonly) |
| cb_person_cred_hist_length | number | ≥ 0 |
| credit_score | number | 300–850 |
| previous_loan_defaults_on_file | select | Yes, No |

#### Особенности вёрстки

- **Адаптивная сетка**: на десктопе поля расположены в две колонки, на мобильных устройствах схлопываются в одну.
- **Семантические теги**: используется `<form>`, `<label>`, `<input>`, `<select>`.
- **Доступность**: все поля имеют подписи, корректный tab-порядок.

### style.css — Стилизация интерфейса

Файл стилей реализует **две цветовые темы** (светлая и тёмная) с использованием CSS-переменных, что позволяет легко поддерживать обе темы без дублирования кода.

#### Архитектура стилей

```css
:root {
  /* Светлая тема (по умолчанию) */
  --bg-gradient-start: #e6f0ff;
  --bg-gradient-end: #d4e4fc;
  --card-bg: rgba(255, 255, 255, 0.9);
  /* ... */
}

body.dark-theme {
  /* Тёмная тема — переопределение тех же переменных */
  --bg-gradient-start: #0b1120;
  --bg-gradient-end: #0f172a;
  --card-bg: rgba(15, 23, 42, 0.75);
  /* ... */
}
```

#### Ключевые компоненты

| **Компонент** | **Описание** |
|---------------|--------------|
| **Glass-карточки** | Полупрозрачные карточки с размытием фона (backdrop-filter: blur) |
| **Плавные анимации** | Трансформации при наведении, переходы между темами |
| **Спиннер загрузки** | Вращающаяся анимация при ожидании ответа от сервера |
| **Бейджи статуса** | Зелёный («Одобрено») и красный («Отказано») для визуализации результатов |
| **Адаптивность** | Медиазапросы для планшетов и мобильных устройств |

#### Используемые шрифты

- **Основной шрифт**: Inter (Google Fonts) — современный гротеск с хорошей читаемостью
- **Размеры**: от 0.7rem (подписи) до 2rem (заголовки)

### script.js — Клиентская логика

Основной JavaScript-файл, отвечающий за взаимодействие с пользователем, отправку запросов к API и обработку ответов.

#### Архитектура кода

```
┌─────────────────────────────────────────────┐
│             КОНФИГУРАЦИЯ                     │
│  - USE_MOCK (true/false)                    │
│  - API_BASE (адрес бэкенда)                 │
└─────────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────────┐
│           ВСПОМОГАТЕЛЬНЫЕ ФУНКЦИИ           │
│  - showLoading() / hideLoading()            │
│  - showError()                              │
│  - getFormData()                            │
│  - displaySinglePrediction()                │
└─────────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────────┐
│             MOCK-ЗАГЛУШКИ                    │
│  - mockPredict()                            │
│  - mockPredictFromCsv()                     │
└─────────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────────┐
│          РЕАЛЬНЫЕ API-ВЫЗОВЫ                │
│  - realPredict() → POST /predict            │
│  - realPredictFromCsv() → POST /predict-from-csv │
└─────────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────────┐
│          ОБЩИЕ ОБЁРТКИ                      │
│  - predictLoan()                            │
│  - predictCsv()                             │
└─────────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────────┐
│         ОБРАБОТЧИКИ СОБЫТИЙ                 │
│  - submit → /predict                        │
│  - click → /predict-from-csv                │
│  - toggle → смена темы                      │
└─────────────────────────────────────────────┘
```

#### Реализованные функции

**1. Управление темой**

```javascript
function initTheme() {
  const savedTheme = localStorage.getItem('theme');
  // Если сохранена светлая — применяем, иначе тёмная
}
function toggleTheme() {
  // Переключаем класс dark-theme, сохраняем выбор
}
```

**2. Сбор данных из формы**

Функция `getFormData()` собирает значения всех полей, приводит их к нужным типам (числа преобразует в `float` или `int`) и возвращает объект, готовый к отправке на сервер.

**3. Автоматический расчёт loan_percent_income**

```javascript
function updateLoanPercent() {
  const income = parseFloat(incomeInput.value);
  const amount = parseFloat(amountInput.value);
  if (!isNaN(income) && !isNaN(amount) && income > 0) {
    const percent = amount / income;
    percentInput.value = percent.toFixed(4);
  }
}
```

Поле `loan_percent_income` вычисляется как отношение суммы кредита к годовому доходу и обновляется при изменении любого из этих полей. Это исключает ошибки пользователя.

**4. Отправка одиночного предсказания**

При отправке формы:
- Собираются данные → показывается спиннер → отправляется POST-запрос на `/predict`
- Ответ отображается в виде цветного бейджа (Одобрено/Отказано)
- Скрывается спиннер

**5. Обработка CSV-файла**

При нажатии кнопки «Загрузить и предсказать»:
- Выбранный файл читается через FileReader
- Отправляется на сервер через `multipart/form-data`
- Отображаются ROC-AUC и таблица с предсказаниями

**6. Отображение таблицы**

Таблица строится динамически на основе полученных данных: заголовки извлекаются из ключей объекта, строки заполняются соответствующими значениями.

**7. Обработка ошибок**

Все запросы обёрнуты в `try/catch`:
- Ошибки сети (сервер недоступен)
- Ошибки валидации (422)
- Ошибка «Модель не загружена» (503)

Все ошибки выводятся пользователю через `alert()` с понятным описанием.

## 4. Интеграция с API

### Согласованный API-контракт

Перед началом разработки был согласован формат запросов и ответов с Backend-разработчиком:

#### POST /predict

**Запрос:**
```json
{
  "person_age": 35,
  "person_gender": "male",
  "person_education": "Bachelor",
  "person_income": 75000,
  "person_emp_exp": 5,
  "person_home_ownership": "rent",
  "loan_amnt": 200000,
  "loan_intent": "homeimprovement",
  "loan_int_rate": 4.5,
  "loan_percent_income": 0.27,
  "cb_person_cred_hist_length": 8,
  "credit_score": 710,
  "previous_loan_defaults_on_file": "No"
}
```

**Ответ:**
```json
[
  {
    "person_age": 35,
    "person_income": 75000,
    // ... все остальные поля
    "loan_status": "approved"
  }
]
```

#### POST /predict-from-csv

**Запрос:** `multipart/form-data` с файлом `.csv`

**Ответ:**
```json
{
  "roc_auc": 0.8745,
  "data": [
    { "person_age": 22, "loan_status": 1, "predicted_loan_status": "approved", ... },
    { "person_age": 45, "loan_status": 0, "predicted_loan_status": "rejected", ... }
  ]
}
```

### Режимы работы

| **Режим** | **Переменная** | **Описание** |
|-----------|----------------|--------------|
| Mock-режим | `USE_MOCK = true` | Заглушки имитируют ответы модели (для разработки без бэкенда) |
| Реальный режим | `USE_MOCK = false` | Запросы отправляются на реальный бэкенд |

В mock-режиме:
- `/predict` принимает решение на основе простого правила (доход > 50000 и рейтинг > 650)
- `/predict-from-csv` генерирует случайный ROC-AUC и добавляет случайные предсказания

### Обработка CORS

Для корректной работы с бэкендом на разных портах бэкенд-разработчик настроил CORS в FastAPI. Это позволило фронтенду, открытому через `file://` или `localhost:8080`, отправлять запросы на `localhost:8000`.

## 5. Работа в команде

### Распределение задач

Для параллельной разработки мы использовали следующий подход:

1. **Согласование API-контракта** — первым делом мы с бэкендером определили формат данных.
2. **Mock-заглушки** — я реализовал имитацию работы модели, чтобы не зависеть от готового бэкенда.
3. **Параллельная разработка** — пока бэкендер писал API, я разрабатывал интерфейс и логику.
4. **Интеграция** — после того как бэкенд был готов, мы переключились на реальные запросы.
5. **Тестирование** — совместно проверили все эндпоинты и граничные случаи.

### Коммуникация

- Регулярные синхронизации в чате
- Общий репозиторий с кодом
- Swagger-документация бэкенда (`/docs`) для самостоятельного тестирования

## 6. Трудности и их решение

| **Проблема** | **Решение** |
|--------------|-------------|
| Невозможно разрабатывать без бэкенда | Реализовал mock-функции, имитирующие ответы API |
| Дизайн выглядел несовременно | Переработал дизайн в стиле glassmorphism с синей гаммой |
| В тёмной теме заголовок был нечитаем | Использовал яркий бело-голубой градиент специально для тёмной темы |
| Пользователь мог ошибиться в расчёте `loan_percent_income` | Добавил автоматический расчёт при изменении дохода или суммы кредита |
| Некорректная работа с типами данных | Использовал `parseFloat()` и `parseInt()` с проверками `isNaN()` |
| Переключение темы не сохранялось | Добавил сохранение выбора в `localStorage` |

## 9. Запуск проекта

### Локальный запуск

1. Убедитесь, что бэкенд запущен на `http://localhost:8000`
2. Откройте папку `frontend/` в терминале

**Способ 1 — через live-server (рекомендуется):**
```bash
npm install -g live-server
live-server
```

**Способ 2 — просто открыть в браузере:**
```bash
start index.html
```

### Переключение на реальный бэкенд

В файле `script.js` изменить:

```javascript
const USE_MOCK = false;   // было true
const API_BASE = 'http://localhost:8000'; // адрес бэкенда
```

### Загрузка модели

Перед использованием интерфейса необходимо загрузить модель через Swagger UI:
1. Открыть `http://localhost:8000/docs`
2. POST `/upload-model` → выбрать `pipeline.pkl` → Execute
